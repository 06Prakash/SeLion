PageYAML for iOS automation

This section is focused on SeLion PageYAML to facilitate automation for iOS apps. Concepts outlined in this section build on those described in SeLion PageYAML for web flow.

Contents of a PageYAML File

A PageYAML file that represents a page (or screen) in an iOS application is made up of the following parts.

  • baseClass - The value for this key represents the fully qualified package name of a class that will serve as the base class to inherit attributes and/or capabilities from. If your page is new and is not customizing an existing page (for example -- you may have a login page which extends a header page), then you should specify the value com.paypal.selion.testcomponents.IOSPage which by default is the class within SeLion that provides all the basic operations that are required for any given iOS page, including instantiation logic.
  • platform - The value for this key represents the platform the YAML is intended for. By default, if nothing is specified it is taken as "web". For platforms like iOS it must be specified as "ios".
  • elements - The value for this key is a set of key/value pairs that represent elements in a page.
    • The name to be given to identifying the locator/selector. This name would end up serving as the data member in the generated class and maps to the element specified. The rules for naming locators/selectors are as follows:
      • <prefix>UIAButton Is a Button UI element.
      • <prefix>UIALink Is a Link UI element.
      • <prefix>UIANavigationBar Is a Navigation Bar UI element.
      • <prefix>UIASlider Is a Slider UI element.
      • <prefix>UIAImage Is a Image UI element.
      • <prefix>UIATextField Is a TextField UI element.
      • <prefix>UIAStaticText Is a Text UI element.

      Here prefix is a meaningful name that you would give to your element mapping it to the element on your page.

      • Note: Depending on the platform, UI element names are different. For example; On a web platform a button is located by //button and has a name with the word Button in it. On the iOS platform a button is located by //UIAButton and has a name with the word UIAButton in it.

      The list of supported iOS elements can be found here

    • locators: - This key represents the set of values that map to the element name.
    • Localized locator - Values for the locators/selectors which are comprised of a key/value pairs -- the locale the element will exist in and the actual locator/selector to find the element by. For example; english: "//UIAStaticText[contains(@name,'climbed')]". This equates to -- For the english version of the app, there should exist a UIAStaticText element whose name contains climbed on the page/screen this file represents.
