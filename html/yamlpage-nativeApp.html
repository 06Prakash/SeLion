Customizing Yaml Page for iOS Native App automation

The section on SeLion PageYaml outlines details on using Yaml for automating Web Flow. This section is focused on customizing SeLion Page Yaml to facilitate automation for iOS Native Apps.

Customized contents of a Yaml Page

A Yaml page that represents page related information, would essentially have the below mentioned contents in it:

  • baseClass - The value for this key represents the fully qualified package name of a class that will serve as the base class from which all common abilities that any page in the SeLion world for a Native App would possess. If your page is totally new and is not customizing an existing page (for e.g., you may have a login page which perhaps extends a header page ), then you should be specifying the value for this key as com.paypal.selion.testcomponents.IOSPage because by default this is the class within SeLion that provides for all the basic operations that are associated with a given page, including logic pertaining to instantiation of the page as well.
  • platform - The value for this key represents the platform to which the data in the yaml file works for. By default, if nothing is specified it is taken as "web". For platforms like iOS it must be specified as "ios".
  • elements - The value for this key is a set of key/value pairs that represents the below information pertaining to html elements in the page.
    • The name to be given to identifying the locator. This name would end up serving as the data member in the generated class which represents this element. The rules for naming the locator are as below :
      • <prefix>UIAButton Is a Button UI element.
      • <prefix>UIALink Is a Link UI element.
      • <prefix>UIANavigationBar Is a Navigation Bar UI element.
      • <prefix>UIASlider Is a Slider UI element.
      • <prefix>UIAImage Is a Image UI element.
      • <prefix>UIATextField Is a TextField UI element.
      • <prefix>UIAStaticText Is a Text UI element.

      Here prefix is a meaningful name that you would give to your element mapping it to the html element on your page.

      • Note: The UI element names are different for platforms. Ex: On a web platform its Button where as on a iOS Platform its UIAButton.The <prefix> must be followed by the element name as appropriate to the platform.

      The list of supported IOSeLionElement can be found here List of elements

    • locators: - This key represents the set of values that map to the element name.
    • values for the locators which represents a key/value pair of locale and the actual locator for the html element.
