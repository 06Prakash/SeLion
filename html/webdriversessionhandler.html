Understanding WebDriverSessionHandler

WebDriverSessionHandler is a utility to manage a particular WebDriver session when a test mixes web and non-web operations.

By default, the WebDriver session will timeout based on the timeout value specified in the grid configuration. WebDriverSessionHandler is useful in cases where you are performing a non-web operation in your test case (i.e. an API call, a SQL operation, or other) that may cause your WebDriver session to timeout -- often throwing a RuntimeException with the message "Session timed out." WebDriverSessionHandler helps to overcome this problem by keeping the WebDriver session alive.

How WebDriverSessionHandler Works

WebDriverSessionHandler has two methods:

  • start() - Begins polling the current WebDriver session to prevent it from timing out.
  • stop() - Stops the polling.

The steps look something like this:

  • Create an instance of WebDriverSessionHandler by passing it a reference of the current test's WebDriver instance.
  • Invoke the start() method on it. This causes the WebDriver session to not expire
  • Include all your non-web operations.
  • Invoke the stop() method to instruct WebDriverSessionHandler to stop polling the web browser.
