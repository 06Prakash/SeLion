Mobile Automation

How to create a mobile test using SeLion.

Prerequisites

Make sure the following software is installed before you start:

  1. Java JDK 1.7
  2. Maven 3
  3. Xcode 5.1.1+ (Mac installs only, required for iOS tests) or the Android SDK (for Android tests)
    • For full details on how to setup your Mac for iOS testing with ios-driver refer to the project site.
    • For full details on how to setup your machine for Android testing with Selendroid refer to the project site.

Note: the example below is iOS oriented. However, after you complete the prerequisites, the approach is the same. Android specific documentation is in the works.

Additional Considerations for a @MobileTest

  
As seen from the above sample, a test can specify the application that it is trying to automate.
    There are a couple of things that can be specified via the @MobileTest annotation. For example:

    
    
      
  1. appName - The native application against which the automation test needs to be run.
    2. 
      
  2. device - The device type to be used. E.g. - an iPhone, an iPad, an Android device, etc
    3. 
      
  3. language - The application's language that is to be used.
    4. 
      
  4. locale - The application's locale that is to be used.
    5. 
    

  
Refer to the SeLion client javadocs for the complete list of annotation parameters.


  
Some of the defaults attributes for the @MobileTest annotation can be influenced via Java
    system properties, environment variables, or via TestNG suite parameters. 

    

      
      

        
          TestNG Suite parameter
        
        
          JVM System Property / Environment Variable
        
        
          Description
        
      

      
      
      

        appFolder
        SELION_SELENIUM_NATIVE_APP_FOLDER
        Represents the location of a folder which contains the mobile applications (.app and .apk files) under test.
          This is applicable only for local runs.
      

      

        appName
        SELION_SELENIUM_NATIVE_APP_NAME
        Override the default name of the application that is to be used for running tests. If nothing is specified, it
          defaults to ""
      

      

        appLanguage
        SELION_SELENIUM_NATIVE_APP_LANGUAGE
        Override the default language to be used for running tests. If nothing is specified, it defaults to
          en (English)
      

      

        appLocale
        SELION_SELENIUM_NATIVE_APP_LOCALE
        Override the default locale to be used for running tests. If nothing is specified, it defaults to
          en_US (US English)
