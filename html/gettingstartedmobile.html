Getting Started with Mobile Automation

How to create a mobile test using SeLion.

Prerequisites

Make sure the following software is installed before you start:

  1. Java JDK 1.7
  2. Maven 3
  3. Xcode (Mac installs only, required for iOS tests)
  4. SeLion installed into your local Maven Repository

For full details on how to setup your Mac to work with ios-driver refer to the ios-driver project site.

Additional Considerations for a @MobileTest

  
As seen from the above sample, a test can specify the application that it is trying to automate.
    There are a couple of things that can be specified via the @MobileTest annotation. They are:

    
    
      
  1. appName - The native application against which the automation test needs to be run.
    2. 
      
  2. device - The device type to be used. For e.g., an iPhone (or) an iPad.
    3. 
      
  3. language - The application's language that is to be used.
    4. 
      
  4. locale - The application's locale that is to be used.
    5. 
    


  
All the above mentioned attributes of the @MobileTest annotation can also be provided via either JVM
    arguments (or) via TestNG suite parameters. Please refer to the below table:

    

      
      

        
          TestNG Suite parameter
        
        
          JVM argument
        
        
          Description
        
      

      
      
      

        <parameter name="appFolder" value="" />
        SELION_SELENIUM_NATIVE_APP_FOLDER
        Represents the location of the folder which contains the Native App. This is applicable only for local runs.
      

      

        <parameter name="appName" value="" />
        SELION_SELENIUM_NATIVE_APP_NAME
        Represents the name of the application that is to be used for running the tests against.
      

      

        <parameter name="appLanguage" value="" />
        SELION_SELENIUM_NATIVE_APP_LANGUAGE
        Represents the language to be used for running the tests against. If nothing is specified it defaults to en (English) 
      

      

        <parameter name="appLocale" value="" />
        SELION_SELENIUM_NATIVE_APP_LOCALE
        Represents the locale to be used for running the tests against. If nothing is specified it defaults to en_US (US English)
